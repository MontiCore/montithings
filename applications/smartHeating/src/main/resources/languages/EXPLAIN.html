Smart Heating

This project simulates an adjustable heating system. It contains following controllable Components:

Example Data Generator

This component exists just for simulation and would not exist in a real implementation. It allows the end-user to adjust the current time, whether or not the time should run, what kepad inputs have been pressed, the state of window and movment sensors.

Movement Sensor

This component allows the end-user to specify which thermostat a movement sensor should activate, as well as for how longit should be activated an at what threshold this should happen.

Temperature Adjustment Unit

This component allows the end-user to specify different modes, which overwrite the temperature setting of the thermostat for a definable time when a specific keypad inpout is made.

Thermostat

This component allows the end-user to specify in which month this thermostat should be turned on, as well as which temperature should be active at what time at which day of the week. It is also possible to activate certain heating periods only if movemnt has been detected.

Notes

  • Inorder to keep the simulation simple time works different in this project. Every two seconds an hour passes and every month is only a week long. There is no smaller increment of time than an hour.
  • Inorder to get any outputs of this simulation you need to read the logs of the Example Data Printer: "docker logs -f smartHeating.heatingSystem.edp"
