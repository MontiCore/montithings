Thermostat

This component defines when and with which temperature under what circumstances the theoratically connected radiator should heat.

Model Format Rules

  • Every valid model needs to define following values in this order: "Month", "Days"
  • Month: This line defines all month in which the system should heat. These are to be defined in a comma seperated list.
  • Days: This line defines at which day and time which temperature shoulkd be used to heat. This is done with a comma seperated list of entries for each relevant day of the week. These entries must look like this. "(<day of the week> -> <start time>-<end time> <optional [M]>: <temperature>, <start time 2>-<end time 2> <optional [M]>: <temperature 2>)" where the optional value "[M]" stands for wether or not htis rule shoudl only be applied when movement has beed detected. For each day you can define an arbitrary amount of rules in a comma seperated list.
